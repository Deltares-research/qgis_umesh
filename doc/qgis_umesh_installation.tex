\section{Installation}
\phantom{mooiman}\vspace{-\baselineskip}
\begin{action}
\item Copy the file \file{\qumesh.dll} to the \Cpp plugin directory of QGIS. 
In my development installation it is:
\dir{c:\textbackslash OSGeo4W64\textbackslash apps\textbackslash qgis\textbackslash plugins\textbackslash qgis\_umesh.dll}.

\item Install the user manual on the program files directory \dir{c:\textbackslash Program Files\textbackslash Deltares\textbackslash qgis\_umesh\textbackslash doc}
\begin{itemize}
	\item qgis\_umesh\_um.pdf
\end{itemize}

\item Install the icons on the program files directory \dir{c:\textbackslash Program Files\textbackslash Deltares\textbackslash qgis\_umesh\textbackslash icons}
\begin{itemize}
    \item file\_open.png
    \item map\_output.png
    \item plotcfts.png
    \item remote\_olv.png
    \item \ldots
\end{itemize}
\item Install the program \plotcfts on the program files directory \dir{c:\textbackslash Program Files\textbackslash Deltares\textbackslash plotcfts}
\end{action}

\paragraph*{Some notes}
\phantom{mooiman}\vspace{-\baselineskip}
\begin{itemize}
\item The \qumesh.dll need the following dll's, so they should be in the \%PATH\%-environment variable:
\begin{itemize}
    \item qt5core.dll
    \item qt5gui.dll
    \item qt5widgets.dll
\end{itemize}

\item Next dll's are probably already placed in the \%PATH\%-environment variable when installing QGIS:
\begin{itemize} 
    \item qgis\_core.dll
    \item qgis\_gui.dll
\end{itemize} 
\end{itemize}
